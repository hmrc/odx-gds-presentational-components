# Form Group

Infrastructure component used to simplify label, hint, error handling for inputs.

The form group component expects a single 'input element' as a child and supporting props ```label```, ```name``` and ```id```
The FormGroup component will then add the name and id attributes to the input child, and handle the linking of the lable with the 'for' attribute,
and passed ```hintText``` and ```errorText``` text with the aria-describedby attribute, internally

Other props can be passed to the form group to control the label display: ```labelIsHeading``` and ```labelClasses```
You can also override the hidden text prefixed to the error message, in case of need for translation with ```errorPrefix``` (This value defaults to 'Error:' if not provided)

## Details

Form group component defaults to setting its label as page header
<Canvas of={FormGroupStories.Default}/>
But can be set to not do this, by setting labelIsHeading as false
<Canvas of={FormGroupStories.LabelNotHeading}/>

The Form group can accept either a react node or a functional component as children.
The Form group Component will attempt to manage the aria-describedby ids for its children if there is hint or error text.

### A note about handling of describedby ids
The most reliable way to do this is pass a Functional Component, which receives an object as props (formgroupprops).
The component should then spread these props in the input component
e.g.
```(formGroupProps) =>  <wrapper><input {...formGroupProps} /></wrapper>```
This is especially useful when the input is wrapped or has prefixes/suffixes.

Alternatively, if children is a reactElement, the Form group will attempt to clone it and append the aria-describedby attributes, however this
will only correctly work if the element is a pure input.
