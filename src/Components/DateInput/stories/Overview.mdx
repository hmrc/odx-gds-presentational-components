## Details

The Date input will  accept autocomplete property, assigning relevant autocomplete value to each field
(e.g. passing bday will assign relevant suffix for each field, bday-day, bday-month, bday-year)
<Canvas of={DateInputStories.Autocomplete}/>

Each field can be passed it's own changeHandler in the field data (e.g. ```dayField={onChange:()=>{}```)
The value of each individual field is passed with it's field data (e.g. ```dayField={value:01}```)
<Canvas of={DateInputStories.WithDefaultValue}/>

Each individual field name can be overriden, to enable translation for example (e.g. ```dayField={label:'Ddyd'}```)
<Canvas of={DateInputStories.OverridenFieldNames}/>

By default, an error will cause all inputs to be highlighted:
<Canvas of={DateInputStories.GeneralError}/>
If the error only targets a specific field(s) that can be configured with field data (e.g. ```monthField={hasError=true}```)
<Canvas of={DateInputStories.FieldSpecificError}/>


**The date input uses the fieldset component, and so any fieldset props can also be passed via the Date Inuput**
[Fieldset documentation](/docs/gds-components-field-set--overview)
